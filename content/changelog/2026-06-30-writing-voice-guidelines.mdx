`WRITING_VOICE.md` got a stricter pass. The rules now make the site's copy expectations easier to apply across articles, UI text, docs, bios, and page descriptions.

The point is not to make every sentence sound identical. It is to keep the site out of generic product-marketing language and closer to the way I actually explain the work: direct, first-person when appropriate, plain, and specific.

The agent memory docs were synced too, so future edits start from the same guidance instead of rediscovering the voice rules file by file.
